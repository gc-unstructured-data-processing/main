v0.4 한 페이지 요약 — 이동원 솔로 + 작업별 보고서

v0.2 ROADMAP(4명 분업)이 작업 분담 측면에서는 깔끔했지만, 실제로는 이동원 1인이 전 단계를 끌고 가게 됨. v0.4는 이 현실을 반영하여 모든 단계의 산출물·보고서·버전 관리를 이동원 책임으로 재정렬. 통합 단계(_integrated/)는 추후 별도 1인.

이 ROADMAP의 사용 방식 — 작업 시작 전 해당 단계 카드를 한 번 읽고, 산출물 매트릭스에서 다음 파일명을 확인. 끝나면 위반금지 8항목 self-check + 보고서 마무리.

v0.2 → v0.4 변경의 핵심 4가지

1 · 솔로 워크플로우

4명 분업 → 1인 책임

  • v0.2의 A·B·C·D 역할 매트릭스 deprecate
  • 이동원이 수집·전처리·정제·마이닝 전 단계 작업
  • 모든 작업물은 members/이동원/ 안에
  • 통합은 5/19 이후 별도 1인 (이동원 또는 별도)

2 · 작업별 보고서

교수님 가이드 03 형식
  • 각 노트북당 1개 보고서 (.md)
  • 양식: 목적 / 데이터 / 방법 / 결과 / 한계 / 다음 단계
  • 작성 시점: 셀 단위 메모 (Q4-a) + 마무리 정리 (Q4-c)
  • 파일명: {NN}_{단계}_phase{N}_v{M}_보고서.md

3 · 노트북 버전 명시

v→v+1 모두 보존
  • 명명: {NN}_{단계}_phase{N}_v{M}.ipynb
  • 예: 01_collection_phase1_v1.ipynb → v2 → v3
  • 사소한 수정(주석·변수명)도 새 파일 (rule 1: 버전 보존)
  • 구버전 삭제 ❌ — 결과 안 좋으면 rollback

4 · 웹크롤링·벤치마킹 지속

작업마다 1건 이상
  • 각 작업마다 외부 코드 1건 + 학술 논문 1건 이상 확인
  • 벤치마킹 결과는 보고서 "방법" 섹션에 인용
  • v0.2 COLLECTION 4-track 벤치마킹 그대로 유지 (강의·공개코드·실무·논문)
  • 웹크롤링은 가이드 매트릭스 안에서만 (DART API 주력)
